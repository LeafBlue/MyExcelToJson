该项目使用C++qt语言开发，版本5.15.2 mingw。
该项目使用的库有 QXlsx 以及 nlohmann::json库。

该项目采用扁平化格式转换json。这种设计的优点是无需对excel进行复杂的设计，能够在前期决定表头即可安心配置。
该项目采用递归的方式逐层解析，解析的深度取决于栈空间，基本能够覆盖大多数需求。

关于解析的规则说明：
第一行，填充字段名
第二行，填充对应的数据类型，支持int、str、json三种格式，主要是用来决定生成json时是否在数据两侧加引号。
第三行，填充注释，如果不想填写注释，也至少要有1个单元格内有数据，不允许完全留空，否则会中断Excel的读取。

从第四行开始，填充数据，数据要对应上方的字段名。

关于字段名的说明：
字段名主要支持以下几种类型：
1 普通字段名
例如 id、name。
这样的字段名，只要将数据填入下方单元格即可。

2 数组结尾的字段名
例如 objs[]
这样的字段名，需要将一组数据填入一个单元格，并且使用英文逗号“,”隔开。
无论数据类型是数值还是字符串，都不需要加引号，会根据第二行的内容自动解析。

3 字段名中存在点“.”的字段
例如 job.id、job.name，
上述情况，job将会自动展开成为对象，其中，每个属性占据1列，且需要1个单元格进行配置
点“.”不允许出现在结尾，不允许出现后不跟随属性名，后果未知。

4 字段名中存在数组对象“[].”的情况
例如job[].id、job[].name
上述情况中，job将会创建一个数组，内部存在多个对象。
此时，每个对象的每个属性占据1列，且需要1个单元格进行配置。
当数组中存在1个以上对象时，需要另起一行，在对应的列下进行配置
当另起一行时，下一个对象需要下移，不能和当前数组对象重叠配置。

5 嵌套
所谓嵌套，指的是基于上述基本情况的衍生情况。请注意，衍生情况必须符合json常理，不能随意想象出无法转换为json的组合。
上面提到的job.id、job.name，便是最基本的嵌套，嵌套的是基本字段id、name
除此之外还可以有其他嵌套方式，例如
job.name.a
job.time[]
job.day[].first
job[].week[].b
