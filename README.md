# Excel 转 JSON 转换工具说明

##  项目基本信息
- **开发语言**：C++ / Qt
- **框架版本**：Qt 5.15.2 (MinGW)
- **核心依赖**：
  - `QXlsx`（Excel 读写解析）
  - `nlohmann::json`（JSON 序列化与生成）

##  设计理念与解析机制
- **扁平化格式转换**：采用扁平化结构映射 JSON。无需对 Excel 进行复杂的层级排版，只需在前期确定好表头即可快速配置，大幅降低设计成本。
- **递归逐层解析**：采用递归方式深度解析字段。解析深度受限于系统栈空间，足以覆盖绝大多数业务场景。

##  表格解析规则
| 行号 | 内容要求 | 详细说明 |
| :--- | :--- | :--- |
| **第 1 行** | 字段名 (Key) | 定义 JSON 的键名 |
| **第 2 行** | 数据类型 | 支持 `int`、`str`、`json`。用于决定生成 JSON 时数据两侧是否自动添加引号 |
| **第 3 行** | 注释说明 | 可选填。若无需注释，**至少保证 1 个单元格内有数据**，严禁完全留空，否则会导致 Excel 读取中断 |
| **第 4 行起** | 业务数据 | 填充实际数据，需严格对应上方字段名进行配置 |

##  字段名命名规范
字段名主要支持以下 5 种类型，请根据业务结构选择：

### 1. 普通字段
- **示例**：`id`、`name`
- **用法**：直接在下方对应单元格填入数据即可。

### 2. 数组字段（以 `[]` 结尾）
- **示例**：`objs[]`
- **用法**：将一组数据填入**单个单元格**，使用英文逗号 `,` 分隔。
- **注意**：无论数据类型是数值还是字符串，**无需手动添加引号**，程序会根据第二行的类型定义自动处理。

### 3. 对象嵌套字段（含 `.`）
- **示例**：`job.id`、`job.name`
- **用法**：`job` 将自动展开为 JSON 对象。每个属性占 1 列，需 1 个单元格配置。
- **警告**：`.` **严禁出现在字段名末尾**，且其后必须跟随属性名，否则可能导致未知解析错误。

### 4. 对象数组字段（含 `[]`.`）
- **示例**：`job[].id`、`job[].name`
- **用法**：`job` 将创建一个数组，内部包含多个对象。每个对象的每个属性占 1 列。
- **换行规则**：当数组包含多个对象时，需**另起一行**在对应列下配置。换行时，下一个对象的数据需整体下移，**不可与当前数组对象的数据行重叠**。

### 5. 复杂嵌套
- **说明**：基于上述基本规则的自由组合。请遵循 JSON 数据结构常理，避免生成非法的 JSON 结构。
- **基础嵌套**：`job.id`、`job.name`（嵌套基础字段）
- **高级嵌套示例**：
  ```text
  job.name.a
  job.time[]
  job.day[].first
  job[].week[].b
